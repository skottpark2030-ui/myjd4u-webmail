================================================================
  myjd4u.com 배포 패키지 — 폴더 그대로 올리면 됩니다
================================================================

이 패키지는 두 개의 repo로 나뉩니다.
  [1] site/      → 메인 사이트 repo (myjd4u.com, 정적 호스팅)
  [2] webmail/   → 메일 repo (mail.myjd4u.com, 별도 Node 서비스)

각 repo는 서로 다른 Render 서비스입니다. 섞지 마세요.


----------------------------------------------------------------
[1] site/  →  메인 사이트 repo
----------------------------------------------------------------
아래 구조 그대로 repo에 올리거나, 해당 파일만 교체하면 됩니다.

  site/
  ├─ index.html              ← 홈  (※ 아래 '홈 주의' 참고)
  ├─ lib/
  │   ├─ index.html          ← 공개 자료실
  │   └─ files/sample.html   ← 예시 파일 (이미 있으면 둬도 됨)
  ├─ internal/
  │   ├─ index.html          ← 내부 자료실
  │   └─ files/sample.html   ← 예시 파일
  └─ assets/
      ├─ logo-mark.png
      └─ logo-full.png

올린 뒤 Render 자동 재배포(1~2분). 확인:
  https://myjd4u.com/lib/        → 썸네일 카드 + 분류 칩
  https://myjd4u.com/internal/   → 권한 배지(임원/팀/공통)

◆ 홈 주의 (index.html)
  지금 홈의 '열람 신청' 폼은 화면 접수만 합니다(실제 메일 발송 X).
  기존 홈 폼이 실제로 신청 메일을 보내고 있었다면,
  index.html 은 폼 전송을 연결한 뒤 마지막에 올리세요.
  (자료실/내부/메일만 먼저 올려도 사이트는 정상 동작합니다.)
  → home-navy.html 안 <script>의 submit 함수 주석 위치에 전송 코드 연결.


----------------------------------------------------------------
[2] webmail/  →  메일 repo (mail.myjd4u.com)
----------------------------------------------------------------
기존 웹메일 repo의 파일을 아래 내용으로 교체(덮어쓰기):

  webmail/
  ├─ server.js               ← 삭제/휴지통 엔드포인트 추가됨
  ├─ public/webmail.html     ← 휴지통 탭 + 삭제/복원/영구삭제 UI
  ├─ package.json            ← 변경 없음(동일)
  └─ README.md               ← 안내 갱신

  * node_modules 는 넣지 않았습니다 — Render가 npm install 자동 실행.
  * 환경변수(SESSION_SECRET 등)는 그대로 두세요.

올린 뒤 Render 재배포. 확인:
  https://mail.myjd4u.com 로그인 → 상단 '휴지통' 탭 보이면 반영됨
  메일에 마우스 올려 🗑 → 휴지통에서 복원/영구삭제 테스트
